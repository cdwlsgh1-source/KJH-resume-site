# 자동제어 분야 HTML 이력서 제작 — Claude Code 단계별 프롬프트

> PRD를 기반으로 Claude Code(터미널/CLI)에서 순서대로 입력할 프롬프트입니다.
> 각 단계를 순서대로 실행하고, 결과를 확인한 뒤 다음 단계로 넘어가세요.

---

## 사전 준비

```bash
mkdir resume-site
cd resume-site
claude
```

Claude Code 실행 후 아래 프롬프트를 순서대로 입력합니다.

---

## STEP 1. 프로젝트 초기 설정 및 구조 설계

```
자동제어 분야 엔지니어를 위한 HTML 이력서를 만들 거야.
아래 요구사항에 맞는 프로젝트 구조를 잡아줘.

- 단일 index.html 파일 (CSS/JS 내장, 외부 의존성 최소화)
- 반응형 디자인 (데스크톱/모바일 대응)
- 인쇄 시(@media print) 깔끔하게 출력되는 스타일 포함
- 미니말·화이트 톤 디자인, 여백 넣낙, 가독성 높은 타이포그래피
- 포인트 컴러는 네이비/블루 계열 1가지만 사용
- 시맨틱 태그(header, section, article 등) 사용, 웹 접근성 고려

먼저 index.html의 기본 붼대(구조)만 만들어줘. 콘텐츠는 다음 단계에서 채울 거야.
```

---

## STEP 2. 섹션별 마크업 구성

```
이제 아래 섹션들을 순서대로 HTML에 추가해줘. 실제 텍스트는 플레이스홀더로 넣어두고,
각 섹션은 id를 붙여서 나중에 스타일링/수정하기 쉽게 만들어줘.

1. 헤더: 이름, 직무 타이틀(자동제어 엔지니어), 연락처(이메일/전화/링크드인)
2. 자기소개(Summary): 3~4줄 핵심 역량 요약
3. 핵심 역량/스킬: PLC(Siemens, Mitsubishi, LS산전 등), SCADA, HMI, 계측기,
   제어 프로그래밍(Ladder, ST 등), 관련 툴/자격증
4. 경력사항: 회사명, 직책, 재직기간, 담당 프로젝트 및 성과(정량적 수치 포함)
5. 프로젝트 하이라이트: 대표 자동화 설비 구축/개선 사례
6. 학력
7. 자격증/교육이수
8. 기타(수상, 특허, 어학 등)
```

---

## STEP 3. 실제 콘텐츠 입력

```
이제 플레이스홀더를 실제 내 정보로 교체할 거야. 아래 내용을 순서대로 불러줄 테니
각 섹션에 맞게 반영해줘.

[여기에 이름, 연락처, 자기소개, 경력, 스킬, 학력, 자격증 등을
 하나씩 불러주며 진행 — 준비되는 대로 항목별로 입력]
```

> 💡 이 단계는 한 번에 다 넣지 말고, "경력사항부터 불러줄게" 식으로
> 섹션 단위로 나눗서 진행하면 더 정확하게 반영됩니다.

---

## STEP 4. 스타일링 (미니말·화이트 톤)

```
콘텐츠가 다 들어갔으니 이제 디자인을 다듭어줘.

- 화이트 배경 + 다크 그레이 텍스트 + 네이비 포인트 컴러
- 섹션 간 여백을 넘낙하게 주고, 타이포그래피 위계(제목/부제목/본문)를 명확히
- 스킬은 태그(chip) 형태나 아이콘+텍스트 형태로 시각화
- 데스크톱에서는 2단 레이아웃(사이드바+메인) 또는 깔끔한 1단 레이아웃 중
  더 어울리는 방식 제안해줘
- 시스템 폰트 스택 사용 (로딩 속도 최적화)
+ 라이트모드 / 다크모드 기능 추가
```

---

## STEP 5. 반응형 대응

```
모바일 화면(360px~480px 기준)에서도 레이아웃이 깨지지 않도록 반응형을 적용해줘.
- 2단 레이아웃이라면 모바일에서는 1단으로 전환
- 폰트 크기, 여백을 모바일에 맞게 조정
- 터치 스크롤이 자연스럽도록 확인
```

---

## STEP 6. 인쇄용(PDF 출력) 스타일 추가

```
이 이력서를 브라우저에서 '인쇄 > PDF로 저장' 했을 때도 깔끔하게 나오도록
@media print 스타일을 추가해줘.

- 불필요한 배경색/그림자 제거 (잃크 절약)
- A4 용지 기준 여백 최적화
- 페이지가 넘어갈 경우 섹션이 중간에 잘리지 않도록 break-inside: avoid 적용
- 링크 URL이 있다면 인쇄 시에도 텍스트로 표시되도록 처리
```

---

## STEP 7. 최종 점검 및 접근성 검토

```
완성된 index.html을 최종 점검해줘.

- 시맨틱 태그와 alt 텍스트, aria-label 등 접근성 요소 누락 없는지 확인
- 폰트 대비(contrast ratio)가 충분한지 확인
- HTML 유효성(중복 id, 닫히지 않은 태그 등) 검사
- 파일 용량이 너무 크지 않은지(이미지 사용 시 최적화 여부) 확인
- 수정할 부분이 있으면 바로 고쳐줘
```

---

## STEP 8. 배포 준비

```
이 index.html을 채용 사이트에 등록/첨부하거나, 별도 URL로 공유할 수 있게
정적 호스팅용으로 배포하고 싶어.

GitHub Pages(또는 Netlify, Vercel) 중 가장 간단한 방법으로 배포하는 절차를
단계별로 안내해줘. 필요한 설정 파일(예: .gitignore 등)도 같이 만들어줘.
```

---

## 참고: 진행 팁

- 각 단계 실행 후 브라우저로 `index.html`을 열어 눈으로 확인하는 것을 권장합니다.
- STEP 3(콘텐츠 입력)은 한 번에 몰아넣지 말고, 섹션별로 나눠서 대화하듯 진행하면
  Claude Code가 각 섹션의 맥락을 더 정확히 반영합니다.
- 스타일이 마음에 안 들면 STEP 4를 다시 요청하며 "OO 느낌으로 바꿔줘" 식으로
  구체적 레퍼런스(색상 코드, 유사 사이트 등)를 주면 결과가 더 좋아집니다.
